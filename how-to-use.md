# Code Runner 使用说明（How to Use）

本文档说明如何使用当前版本的 **Code Runner** 网站。

## 1. 网站能做什么

- 在线运行：`Python`、`C`、`C++`、`Java`、`JavaScript`、`Matlab（Octave 兼容）`
- 支持语言版本选择（优先从在线 API 拉取，失败时使用内置版本）
- 代码输入框、标准输入框（均带行号、固定高度、可滚动）
- 支持在输入框中按 `Tab` 键插入制表符（不会跳到其他控件）
- 支持上传源代码文件并自动填入“程序代码输入”
- 支持上传文本文件并自动填入“程序标准输入（stdin）”
- 支持上传“运行时文件”，程序可通过执行参数中的文件名读取
- 支持输出文件自动下载（按约定输出标记）
- 提供“如何输入文件/如何输出文件”的多语言示例（随语言切换）

---

## 2. 最基础用法（只运行代码）

1. 选择 **编程语言**
2. 选择 **版本**
3. 在“程序代码输入”中写代码
4. （可选）在“程序标准输入（stdin）”中输入 stdin
5. 点击 **运行代码**
6. 在“输出信息”中查看结果

---

## 3. 执行参数怎么用

- 在“执行参数（可选）”输入框填写命令行参数
- 示例：
  - Python: `input.txt`
  - C/C++/Java/JavaScript: `input.txt --mode fast`

这些参数会传给运行环境，程序可通过各语言的参数接口读取。

---

## 4. 如何上传文件给程序读取

### 方式 A：运行时文件上传（推荐）

1. 在“文件上传（用于程序处理文件）”区域选择文件（如 `input.txt`）
2. 在“执行参数”里填写文件名（如 `input.txt`）
3. 运行代码
4. 程序中按文件名读取

> 页面内已提供“如何编写代码才能输入文件”示例，并会随语言切换。

### 方式 B：标准输入文件导入

1. 在“程序标准输入（stdin）”区域点击 **上传输入**
2. 选择文本文件
3. 文件内容会直接写入 stdin 输入框
4. 程序通过标准输入读取

---

## 5. 如何让程序输出文件并自动下载

仅仅在程序里写本地文件还不够，必须把文件内容按约定打印到输出流。

页面会解析以下标记块并自动触发下载：

```text
__OUTPUT_FILE_BEGIN__
{"name":"result.txt","type":"text/plain","base64":"..."}
__OUTPUT_FILE_END__
```

说明：

- `name`：下载文件名
- `type`：MIME 类型，如 `text/plain`、`application/json`、`text/csv`、`image/png`、`application/octet-stream`
- `base64`：文件二进制内容的 Base64 字符串

> 页面内已提供“如何编写代码才能输出文件”示例，并会随语言切换。

---

## 6. 编辑器小技巧

- 行号已内置在输入框左侧
- 输入内容过长时可通过滚动条浏览
- 按 `Tab` 可插入制表符
- 可使用“上传源代码”快速加载本地代码文件

---

## 7. 常见问题

### Q1：为什么我上传了文件，但程序读不到？

- 请确认执行参数里填写了正确文件名（含扩展名）
- 请确认程序从参数读取的是第一个参数（如 Python 的 `sys.argv[1]`）

### Q2：程序执行成功，但没下载文件？

- 请确认程序输出了 `__OUTPUT_FILE_BEGIN__` 和 `__OUTPUT_FILE_END__` 包裹的 JSON
- 请确认 JSON 中包含 `name`、`type`、`base64` 三个字段

### Q3：出现第三方 API 相关错误怎么办？

- 这是在线执行服务限制或波动导致，可稍后重试
- 生产环境建议自建 Judge0 服务或加后端代理

---

## 8. 安全提示

⚠️ 代码执行依赖第三方 API，请不要上传或输入任何涉密、隐私数据/文件。
